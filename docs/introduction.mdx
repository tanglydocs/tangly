# Tangly

Tangly turns a directory of Markdown into a documentation site you can deploy anywhere static files are served. You get rich MDX components, a themed shell with sidebar and search, OpenAPI rendering, and a build that drops onto Vercel, Cloudflare, or any CDN with no configuration.

It is a thin layer over Astro: your project is just `docs.json` and `*.mdx` files. Tangly does the routing, navigation, prose styling, and asset pipeline.

## What you get

- **A working dev server** with HMR over MDX and `docs.json` (sub-second reload).
- **A typed configuration schema** with JSON Schema for `$schema` autocomplete in your editor.
- **Two themes** out of the box (`tang`, `pith`) with full component shadowing for overrides.
- **MDX components**: callouts, cards, tabs, steps, accordions, code groups, OpenAPI viewers, embedded blocks.
- **OpenAPI auto-routing**: drop a spec into `docs.json`, every endpoint becomes a page.
- **Pagefind search** with Cmd+K modal, indexed at build time, exclusion rules respected.
- **Static-site fast**: cold dev under 2s, builds under 30s for ~100 pages.
- **Adapter auto-detection** for Vercel, Cloudflare Pages, Node, or plain static hosts.

## What you write

A project is just two things:

```
my-docs/
├── docs.json
├── introduction.mdx
├── guides/
│   └── getting-started.mdx
└── images/
    └── hero.png
```

`docs.json` declares navigation; each `.mdx` file becomes a page. Frontmatter handles metadata (`title`, `description`, `icon`, `draft`, `noindex`). [Section defaults](/guides/configuration#section-defaults) cascade. [Embedded blocks](/guides/embedded-blocks) let you pull a labeled fragment from one page into another.

## Mintlify compatibility

Tangly understands `docs.json` and `mint.json` formats. An existing Mintlify project renders without source edits — same components, same nav shape, same frontmatter. Run `tangly migrate` to upgrade a `mint.json` project to `docs.json`. See [migrating from Mintlify](/guides/migrate-from-mintlify).
