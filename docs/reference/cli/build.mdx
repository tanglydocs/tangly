# `tangly build`

Produces a static site at `dist/` (or wherever `--out` points).

## Usage

```bash
tangly build [options]
```

## Options

<ParamField path="--out <dir>" type="string" default="./dist">
  Output directory.
</ParamField>

<ParamField path="--adapter <name>" type="string">
  Force adapter: `vercel` | `cloudflare` | `node` | `static`. Auto-detected when omitted (see
  [Deploying](/guides/deploying)).
</ParamField>

<ParamField path="--base <path>" type="string" default="/">
  Deploy under a subpath like `/docs`.
</ParamField>

<ParamField path="--config <path>" type="string" default="docs.json">
  Alternate docs.json path.
</ParamField>

<ParamField path="--root <path>" type="string" default=".">
  Project root.
</ParamField>

## Build output

```
dist/
├── index.html              # redirects to first nav page
├── introduction/index.html
├── guides/<slug>/index.html
├── images/                 # copied from project root
├── logo/
├── sitemap.xml
├── robots.txt
├── llms.txt
└── llms-full.txt
```

## Performance budgets

- Cold build: under 30 seconds for a 100-page project.
- Per-page MDX compile: under 100ms for typical pages.

If you hit these limits, run `tangly build --analyze` (Phase 2) to get a build report.

## Drafts and noindex

Pages with `draft: true` are omitted entirely. Pages with `noindex: true` are built but excluded from `sitemap.xml`, `llms.txt`, and `llms-full.txt`.

## Including drafts in staging

```bash
TANGLY_INCLUDE_DRAFTS=1 bun x tangly build
```

Drafts ship in the build but still get a "Draft" badge.
