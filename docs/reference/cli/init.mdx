# `tangly init`

Interactive wizard that scaffolds a new Tangly project.

## Usage

```bash
tangly init [dir] [options]
```

If `dir` is omitted, scaffolds into the current directory.

## Options

<ParamField path="--from <dir>" type="string">
  Walk an existing folder of `.md`/`.mdx` files and synthesize a `docs.json` from the tree (folders → groups). Idempotent — re-running merges new files into the existing nav rather than clobbering it. Use this to bootstrap Tangly on top of a vanilla Markdown repo.
</ParamField>

```bash
# Bootstrap from a Markdown corpus.
tangly init ./my-docs --from ../legacy-wiki

# Re-run later to pick up new files. Existing nav order is preserved.
tangly init ./my-docs --from ../legacy-wiki
```

When `--from` is set, the interactive wizard is skipped — Tangly writes `docs.json` and exits. Pages already in `docs.json` keep their positions; new pages are appended to existing groups (matched by folder name) or added under a new group.

## What it does

<Steps>
  <Step title="Asks for a project name">
    Used in `docs.json` as `name`.
  </Step>
  <Step title="Asks for a template">
    `basic` — hello-world, single page.

    `api` — sample with OpenAPI integration.
  </Step>
  <Step title="Writes files">
    `docs.json` and `introduction.mdx`. Existing `package.json` is left alone — install Tangly yourself with `bun add -D tangly` (or your preferred package manager).
  </Step>
</Steps>

## Output

```
my-docs/
├── docs.json
└── introduction.mdx
```

After init, install Tangly and start the dev server:

```bash
cd my-docs
bun add -D tangly
bun x tangly dev
```

## Errors

`tangly init` refuses to overwrite an existing `docs.json`. If you want to migrate an existing Mintlify project, use [`tangly migrate`](/reference/cli/migrate) instead.
