# Code

Tangly uses [Shiki](https://shiki.style) for syntax highlighting — same engine VS Code ships with — through the `rehype-shiki` plugin. Themes are dual: `github-light` follows the page's light mode, `github-dark` follows dark mode. The active theme switches automatically when the user toggles.

## Fenced code blocks

Plain triple-backtick fences work as you'd expect:

```ts
interface User {
  id: string;
  name: string;
}

const u: User = { id: "abc", name: "Nik" };
```

```python
def add(a: int, b: int) -> int:
    return a + b
```

```bash
bun add -D tangly
bun x tangly dev
```

The first word after the opening backticks is the language. Pass an optional second value for a filename or label:

````md
```ts user-service.ts
const fn = () => {};
```
````

## `<CodeGroup>`

Tabbed code blocks for showing the same thing in multiple languages or frameworks.

<CodeGroup>
  ```ts example.ts
  const x: number = 42;
  console.log(x);
  ```

  ```py example.py
  x: int = 42
  print(x)
  ```

  ```bash example.sh
  echo 42
  ```
</CodeGroup>

````md
<CodeGroup>
  ```ts example.ts
  const x: number = 42;
  ```

  ```py example.py
  x = 42
  ```
</CodeGroup>
````

The label after the language is the tab title. The first child is active by default.

## Inline code

Single-backtick `` `inline code` `` renders as monospace with a tinted background.

## Supported languages

Anything Shiki supports. The full list is in the Shiki docs; the most common ones (`ts`, `tsx`, `js`, `jsx`, `python`, `bash`, `sh`, `zsh`, `json`, `yaml`, `toml`, `html`, `css`, `astro`, `mdx`, `md`, `sql`, `rust`, `go`, `swift`, `kotlin`, `java`, `c`, `cpp`, `csharp`, `dockerfile`, `nginx`, `graphql`, `proto`) all work without configuration.

## Highlighting & line ranges

Tangly currently emits Shiki's default output. Per-line highlighting and `// [!code ++]` annotations are on the roadmap but not wired today. If you need them now, [eject](/reference/cli/eject) and configure Shiki directly.

## Codeblock theme override

Set `styling.codeblocks` in `docs.json`:

```json
{
  "styling": { "codeblocks": "system" }
}
```

| Value | Behavior |
| --- | --- |
| `system` (default) | Light/dark follows the page theme. |
| `dark` | Always uses `github-dark`, even in light mode. |
