# Callouts

Six callout components plus a release-block component (`Update`). All are auto-injected — no imports required.

## Variants

<Note>Heads up — important context.</Note>
<Tip>A pro-tip the reader should know about.</Tip>
<Warning>Watch out — this can cause problems.</Warning>
<Info>Neutral, informational note.</Info>
<Check>This worked / passes.</Check>
<Danger>Critical — proceed with caution.</Danger>

```md
<Note>Heads up.</Note>
<Tip>Pro tip.</Tip>
<Warning>Watch out.</Warning>
<Info>FYI.</Info>
<Check>Verified.</Check>
<Danger>Risk.</Danger>
```

None of the six accept props beyond children. Style and icon vary by variant; the underlying `_Callout.astro` shell renders all of them.

## Update

Used in changelogs. Renders a release block with version label, date, and tag chips.

<Update label="v0.0.1" description="2026-04-29" tags={["alpha"]}>
  ## Initial release

  Renders Mintlify projects unmodified.
</Update>

```md
<Update label="v0.0.1" description="2026-04-29" tags={["alpha"]}>
  Body content goes here.
</Update>
```

### Update props

<ParamField path="label" type="string">
  Version or release name. Renders as the heading.
</ParamField>

<ParamField path="description" type="string">
  Subtitle (typically a date in `YYYY-MM-DD` format).
</ParamField>

<ParamField path="tags" type="string[]">
  Optional list of tag chips rendered to the right of the header.
</ParamField>

`tangly add changelog 1.2.0` scaffolds an `Update` block automatically.

## Custom callout colors / icons

Override any callout via [component shadowing](/guides/custom-components):

```
my-docs/
└── theme/
    └── Note.astro     # your override
```

Your override receives the same children prop. The internal `_Callout.astro` is also overridable if you want to retheme all six in one file.
