# Columns

`<Columns>` is a flat grid wrapper that takes any children. Unlike [`<CardGroup>`](/reference/components/card-group), it doesn't assume the children are cards.

## When to use

- Side-by-side images.
- Two paragraphs of comparison prose.
- Tabular data where a real table is overkill.

For card grids, prefer `<CardGroup>` — it has nicer responsive defaults.

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `cols` | `number` | `2` | Column count. Clamped to `1`–`12`. Built-in responsive classes for 1–6; higher numbers fall back to `grid-cols-N`. |

## Examples

### Two columns of prose

```mdx
<Columns cols={2}>
  <div>
    **Self-hosted.** No vendor lock-in. Build static HTML, drop it on any CDN.
  </div>
  <div>
    **Astro under the hood.** Sub-second HMR, builds under 30s for a hundred pages.
  </div>
</Columns>
```

```mdx
<Columns cols={3}>
  ![](/images/tangly-icon.png)
  ![](/images/tangly-icon.png)
  ![](/images/tangly-icon.png)
</Columns>
```
