# Tabs

`<Tabs>` is the wrapper. Its children must be `<Tab>` elements. The wrapper renders a tablist, manages the active panel, and persists the selection across reloads when you give it a `name`.

## When to use

- Platform-specific instructions (macOS / Linux / Windows).
- Same task in multiple languages or runtimes.
- Equivalent code in TypeScript and Python.

For multiple code blocks where you want one UI, use [`<CodeGroup>`](/reference/components/code-group) instead — it picks up filenames and languages automatically.

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `name` | `string` | — | Sync key. Every `<Tabs>` with the same `name` shows the same tab; selection is mirrored on the page and persisted in `localStorage`. |

## Examples

### Basic

```mdx
<Tabs>
  <Tab title="macOS">Run `brew install bun`.</Tab>
  <Tab title="Linux">Run `curl -fsSL https://bun.sh/install | bash`.</Tab>
  <Tab title="Windows">Use WSL, or `powershell -c "irm bun.sh/install.ps1 | iex"`.</Tab>
</Tabs>
```

<Tabs>
  <Tab title="macOS">Run `brew install bun`.</Tab>
  <Tab title="Linux">Run `curl -fsSL https://bun.sh/install | bash`.</Tab>
  <Tab title="Windows">Use WSL, or `powershell -c "irm bun.sh/install.ps1 | iex"`.</Tab>
</Tabs>

### Synced across the page

```mdx
<Tabs name="lang">
  <Tab title="TypeScript">…</Tab>
  <Tab title="Python">…</Tab>
</Tabs>

…prose…

<Tabs name="lang">
  <Tab title="TypeScript">…</Tab>
  <Tab title="Python">…</Tab>
</Tabs>
```

Both blocks switch in lockstep. The selection survives navigation and refresh because the active tab is keyed off `name` in `localStorage`.
