# Embed

`<Embed>` references a labeled block on another page and inlines its content at build time. Useful for sharing prose across guides without copy-paste drift.

This is **not** an iframe wrapper — for `<iframe>` content (videos, sandboxes), use [`<Frame>`](/reference/components/frame) or [`<Video>`](/reference/components/video).

## When to use

- Rate limit tables that appear in multiple guides.
- Setup snippets reused across language-specific tutorials.
- Compatibility matrices kept in one canonical spot.

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `page` | `string` | — | Source page slug, no leading slash. Required. |
| `block` | `string` | — | Block ID — either an auto-generated heading slug, or the ID from an explicit `{#my-id}` marker. Required. |

## How blocks are identified

Either:

1. **Auto-derived from a heading.** A `## Rate limits` becomes block `rate-limits`.
2. **Explicit marker.** Add `{#rate-limits}` after a heading or directly above a paragraph to pin the ID.

The block extends from the marker to the next heading at the same or higher level.

## Example

In `/docs/guides/billing.mdx`:

```md
## Rate limits {#rate-limits}

Free tier: 60 req/min. Pro: 600 req/min. Enterprise: contact sales.
```

In `/docs/api/getting-started.mdx`:

```mdx
<Embed page="guides/billing" block="rate-limits" />
```

## Constraints

- Embedded fragments are rendered as Markdown only — no MDX/JSX components in embedded content.
- Cycles are detected at build time. The error names both files and the embed trail.
