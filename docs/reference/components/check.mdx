# Check

`<Check>` confirms something is in a known-good state. Use it sparingly — overuse turns it into noise.

## When to use

- After a multi-step procedure: "If you see this output, the migration completed."
- Test pass / build success markers in tutorials.
- Reassurance after a [`<Warning>`](/reference/components/warning) earlier on the page.

## Props

None.

## Example

```mdx
<Check>Run `tangly check` and you should see "OK — no issues found."</Check>
```
