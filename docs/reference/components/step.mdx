# Step

`<Step>` is a single entry in [`<Steps>`](/reference/components/steps). It gets its number automatically from sibling order. You can override the number, the title element size, or attach an icon.

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `string` | — | Step heading. Required. |
| `icon` | `string` | — | Optional icon rendered alongside the number circle. |
| `titleSize` | `"p" \| "h2" \| "h3"` | `"p"` | Heading element. Use `h2`/`h3` if the step body is long enough that the step itself needs to anchor in the table of contents. |
| `stepNumber` | `number` | (auto) | Override the rendered number. Useful for resuming numbering across separate `<Steps>` blocks. |

## Example

```mdx
<Step title="Verify the install" titleSize="h3" icon="circle-check">
  Run `tangly --version`. You should see the version you installed.
</Step>
```

## Resuming numbering

```mdx
<Steps>
  <Step title="Install" />
  <Step title="Initialize" />
</Steps>

…prose break…

<Steps>
  <Step title="Run dev" stepNumber={3} />
  <Step title="Build" stepNumber={4} />
</Steps>
```
