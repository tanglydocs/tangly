# ParamField

`<ParamField>` documents one parameter of an API request. The location (`path`, `query`, `body`, `header`) is encoded in which prop carries the parameter name.

## When to use

- Hand-rolled API reference pages where you don't want a full OpenAPI spec.
- Documenting parameters that don't appear in the spec (operator-only, deprecated, internal).
- Augmenting an `<OpenApiEndpoint>` with extra prose-rich fields.

## Props

Pass exactly one of `path` / `query` / `body` / `header` to set the location and the parameter's name.

| Prop | Type | Default | Description |
|---|---|---|---|
| `path` | `string` | — | Parameter is in the URL path. The string is the parameter name. |
| `query` | `string` | — | Parameter is in the query string. |
| `body` | `string` | — | Parameter is in the request body. |
| `header` | `string` | — | Parameter is a header. |
| `type` | `string` | — | Display-only type label (e.g. `string`, `number`, `boolean`, `User`). |
| `required` | `boolean` | `false` | Marks the parameter as required. |
| `default` | `string` | — | Display-only default value. |
| `deprecated` | `boolean` | `false` | Strikes through the name and adds a "deprecated" chip. |

The body of the component is the description (Markdown allowed).

## Examples

### Path parameter

```mdx
<ParamField path="userId" type="string" required>
  The user's UUID. Returned in `GET /v1/me`.
</ParamField>
```

<ParamField path="userId" type="string" required>
  The user's UUID. Returned in `GET /v1/me`.
</ParamField>

### Query parameter with default

```mdx
<ParamField query="limit" type="number" default="20">
  Maximum results per page. Cap is `100`.
</ParamField>
```

<ParamField query="limit" type="number" default="20">
  Maximum results per page. Cap is `100`.
</ParamField>

### Body parameter, deprecated

```mdx
<ParamField body="legacyEmail" type="string" deprecated>
  Use `email` instead. This field will be removed in v2.
</ParamField>
```
