# Accordion

`<Accordion>` is a single collapsible section. For Q&A-style stacks, wrap multiple in [`<AccordionGroup>`](/reference/components/accordion-group).

## When to use

- FAQ-style content where the reader probably only cares about a few entries.
- Hiding long output, advanced flags, or implementation notes.
- Reducing visual weight on reference pages.

For sequential procedures, prefer [`<Steps>`](/reference/components/steps). For tabbed alternatives, [`<Tabs>`](/reference/components/tabs).

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `string` | — | Visible heading. Required. |
| `defaultOpen` | `boolean` | `false` | Render expanded on first paint. |
| `icon` | `string` | — | Icon next to the title. |
| `description` | `string` | — | Subtitle below the title (always visible, even when collapsed). |

## Example

```mdx
<Accordion title="What does `tangly eject` do?" icon="package-open">
  It writes the synthesized Astro project to `.tangly/` so you can edit
  the runtime directly. One-way operation — commit before running it.
</Accordion>
```
