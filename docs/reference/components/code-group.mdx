# CodeGroup

`<CodeGroup>` wraps two or more code blocks and renders them as tabs. Each child's `data-filename` (set via the `title="..."` code-fence attribute) or language hint becomes the tab label.

## When to use

- Same task in multiple languages.
- Showing a config file alongside the code that loads it.
- Side-by-side equivalents that don't read well stacked.

For installation across npm/yarn/pnpm/bun specifically, `<PackageManager>` is purpose-built — no manual tab labels.

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `name` | `string` | — | Sync key. Same `name` on multiple `<CodeGroup>` blocks → matched tabs across the page and across navigations (persisted in `localStorage`). |

## Examples

### Multiple languages

````mdx
<CodeGroup>
  ```ts title="fetch.ts"
  await fetch("https://api.example.com/v1/users")
  ```
  ```py title="fetch.py"
  requests.get("https://api.example.com/v1/users")
  ```
  ```sh title="curl"
  curl https://api.example.com/v1/users
  ```
</CodeGroup>
````

### Synced across blocks

````mdx
<CodeGroup name="lang">
  ```ts
  console.log("hello")
  ```
  ```py
  print("hello")
  ```
</CodeGroup>

… prose …

<CodeGroup name="lang">
  ```ts
  console.log("done")
  ```
  ```py
  print("done")
  ```
</CodeGroup>
````

Picking TypeScript on the first block selects TypeScript on the second.

## Tab label resolution

Per child, in priority order:

1. `data-filename` (from code-fence `title="..."`)
2. `data-language` (the language token after the opening fence)
3. `"Tab N"` fallback
