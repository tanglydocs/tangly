# Cards

Click-through cards in a responsive grid.

<CardGroup cols={2}>
  <Card title="With icon" icon="rocket" href="/quickstart">
    Description text. The whole card is clickable when `href` is set.
  </Card>
  <Card title="Plain" icon="info">
    No href — non-interactive but still styled.
  </Card>
</CardGroup>

```md
<CardGroup cols={2}>
  <Card title="..." icon="..." href="...">
    Description.
  </Card>
</CardGroup>
```

## `<Card>`

<ParamField path="title" type="string">
  Card heading.
</ParamField>

<ParamField path="icon" type="string">
  Lucide or FA icon name. See [Icons](/reference/schema/icons).
</ParamField>

<ParamField path="href" type="string">
  If set, the entire card becomes a link. External URLs open in a new tab.
</ParamField>

<ParamField path="horizontal" type="boolean">
  Switch from vertical (icon top, content below) to horizontal (icon left, content right).
</ParamField>

<ParamField path="img" type="string">
  Image URL — replaces the icon as the visual.
</ParamField>

<ParamField path="color" type="string">
  CSS color for the icon tint. Defaults to the page's primary color.
</ParamField>

## `<CardGroup>`

<ParamField path="cols" type="number" default="2">
  Grid columns. 1–4 work cleanly across breakpoints.
</ParamField>

The grid is responsive — on narrow viewports cards stack vertically regardless of `cols`.

## `<Columns>`

A more flexible grid for arbitrary content (not just cards).

<Columns cols={3}>
  <Note>Column one.</Note>
  <Tip>Column two.</Tip>
  <Warning>Column three.</Warning>
</Columns>

```md
<Columns cols={3}>
  <Note>...</Note>
  <Tip>...</Tip>
  <Warning>...</Warning>
</Columns>
```

<ParamField path="cols" type="number" default="2">
  Number of columns. Capped between 1 and 12.
</ParamField>

`Columns` does not impose styling on its children — they keep their own visual treatment. Use it to lay out anything in a responsive grid.
