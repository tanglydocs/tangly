# Warning

`<Warning>` flags decisions that bite later. Irreversible commands, breaking changes, configuration that looks fine but isn't.

## When to use

- "`tangly eject` is one-way. Commit before running it."
- "Setting `appearance.strict: true` removes the toggle entirely."
- "`/llms-full.txt` can hit several MB on large sites."

For active danger (data loss, security risk), step up to [`<Danger>`](/reference/components/danger).

## Props

None.

## Example

```mdx
<Warning>Deleting `dist/` while `tangly dev` is running breaks HMR until you restart.</Warning>
```
