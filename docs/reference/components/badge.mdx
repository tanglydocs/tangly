# Badge

`<Badge>` renders a short inline label. Five tones cover most use cases. Starlight-style variant names are accepted as aliases so existing markdown ports cleanly.

## When to use

- "New in v0.3" pills next to feature names.
- Status chips in API references (`Beta`, `Deprecated`).
- Severity tags in changelogs.

## Variants

<Badge>default</Badge> <Badge variant="tip">tip</Badge> <Badge variant="warning">warning</Badge> <Badge variant="error">error</Badge> <Badge variant="accent">accent</Badge>

```mdx
<Badge>default</Badge>
<Badge variant="tip">tip</Badge>
<Badge variant="warning">warning</Badge>
<Badge variant="error">error</Badge>
<Badge variant="accent">accent</Badge>
```

### Aliases

Starlight names resolve to the matching Tangly tone, so `<Badge variant="success">` and `<Badge variant="tip">` render identically:

| Alias | Resolves to |
|---|---|
| `note` | `default` |
| `success` | `tip` |
| `caution` | `warning` |
| `danger` | `error` |

<Badge variant="note">note</Badge> <Badge variant="success">success</Badge> <Badge variant="caution">caution</Badge> <Badge variant="danger">danger</Badge>

## Sizes

<Badge size="small" variant="accent">small</Badge> <Badge size="medium" variant="accent">medium</Badge> <Badge size="large" variant="accent">large</Badge>

```mdx
<Badge size="small" variant="accent">small</Badge>
<Badge size="medium" variant="accent">medium</Badge>
<Badge size="large" variant="accent">large</Badge>
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `"default" \| "note" \| "tip" \| "success" \| "caution" \| "warning" \| "danger" \| "error" \| "accent"` | `"default"` | Tone. Aliases resolve as listed above. |
| `size` | `"small" \| "medium" \| "large"` | `"medium"` | Visual size. |
| `text` | `string` | — | Text content. Used when no children are passed. |
