# GlossaryTerm

`<GlossaryTerm>` renders a glossary entry as an inline link with a hover tooltip carrying the definition. The first occurrence of every glossary term on every page is wrapped automatically by the `rehype-glossary` plugin — you rarely write the tag yourself.

## When to write it manually

- Acronyms that share spelling with non-glossary words and got missed by the auto-pass.
- Inline references where you want the hover-definition without the term being bolded.
- Marking up a term's slug differently from the auto-derived one.

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `slug` | `string` | — | Glossary entry slug (matches the heading's auto-id on `/glossary`). Required. |
| `term` | `string` | (children) | Display label. |
| `definition` | `string` | `""` | Tooltip content. Plain text. |
| `href` | `string` | `/glossary#<slug>` | Override the link target. |

## Example

```mdx
The <GlossaryTerm slug="capacity-factor" definition="The ratio of actual output to maximum possible output.">CF</GlossaryTerm> determines the energy estimate.
```

## Disabling on a page

Frontmatter:

```yaml
---
glossary: false
---
```

The auto-wrap pass skips pages with `glossary: false` in frontmatter. See [Authoring → Glossary](/guides/authoring/glossary).
