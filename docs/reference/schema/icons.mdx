# Icons

Tangly uses [Lucide](https://lucide.dev) by default. Mintlify projects often use Font Awesome names — Tangly aliases ~96 of the most common FA names to Lucide equivalents at render time, so existing content renders without changes.

## Configure

```json
{
  "icons": {
    "library": "lucide"
  }
}
```

| Value | Behavior |
| ----- | -------- |
| `lucide` (default) | Lucide names + FA aliases. |
| `fontawesome` | Reserved — same alias map applies today; explicit FA support arrives later. |

## Where icons are used

- Sidebar entries (`frontmatter.icon` + `navigation.groups[].icon`).
- `<Card icon="..." />`, `<Note icon="..." />`, etc.
- Anchors (`navigation.global.anchors[].icon`).
- Tab labels (`navigation.tabs[].icon`).

## Lucide names

All [Lucide icons](https://lucide.dev/icons) work out of the box. Names are kebab-case in Tangly content (the component normalizes to PascalCase internally):

```md
<Card icon="rocket" title="Launch" href="/launch" />
<Note icon="lightbulb">Reusable advice.</Note>
```

## Font Awesome alias map

Tangly maps ~96 common FA names to their Lucide equivalents in `Icon.astro`. Notable entries:

| FA / Mintlify | Lucide equivalent |
| ------------- | ----------------- |
| `circle-check` | `CircleCheck` |
| `circle-info`, `info-circle` | `Info` |
| `circle-exclamation`, `exclamation-triangle`, `triangle-exclamation`, `warning` | `TriangleAlert` |
| `circle-stop` | `CircleStop` |
| `magnifying-glass`, `search` | `Search` |
| `gear`, `cog` | `Settings` |
| `house`, `home` | `House` |
| `xmark`, `close`, `x` | `X` |
| `bolt`, `zap` | `Zap` |
| `book` | `Book` |
| `terminal`, `code` | `Terminal` / `Code` |
| `git`, `git-branch` | `GitBranch` |
| `key`, `lock`, `shield` | `Key` / `Lock` / `Shield` |
| `database`, `server`, `cloud` | `Database` / `Server` / `Cloud` |
| `envelope`, `mail` | `Mail` |
| `external-link` | `ExternalLink` |
| `arrow-right`, `arrow-left`, `chevron-right`, `chevron-down` | matching Lucide arrows |
| `sparkles`, `rocket`, `star`, `heart` | matching Lucide |

The full map lives in `packages/theme-ui/src/components/Icon.astro`. If a name isn't in the map, Tangly falls back to a question mark and emits a console warning.

## Adding your own

Extend the map by [shadowing](/guides/custom-components) the `Icon` component:

```tsx
---
// theme/Icon.astro
import LucideIcon from "lucide/icons/sparkles";
import { Props } from "@tangly/theme-ui/Icon.astro";
const { icon } = Astro.props as Props;
---
{icon === "my-custom" ? <LucideIcon /> : <slot />}
```

## Icon size + color

Components that consume icons usually accept their own props for size/color. The base `<Icon>` component accepts:

```ts
interface Props {
  icon: string;
  size?: number | string;   // default 16
  color?: string;           // CSS color
  class?: string;           // extra class names
}
```
