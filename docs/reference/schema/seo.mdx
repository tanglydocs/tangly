# `seo`, `metadata`, `thumbnails`

Three top-level objects govern search-engine behavior and Open Graph presentation.

## `seo`

```json
{
  "seo": {
    "metatags": {
      "google-site-verification": "abc123",
      "msvalidate.01": "def456"
    },
    "indexing": "navigable"
  }
}
```

| Field | Type | Default | Description |
|---|---|---|---|
| `metatags` | `Record<string, string>` | — | Extra `<meta>` tags injected globally. Key becomes `name` attribute. |
| `indexing` | `"all" \| "navigable"` | `all` | `navigable` excludes orphan pages (not in `navigation`) from `sitemap.xml` and adds `noindex` to them. |

### `indexing: "navigable"`

By default, every `.mdx` file becomes a route — even if no nav entry references it. Setting `indexing: "navigable"` flips that to "only pages reachable from the nav are indexed." Useful if you keep drafts, snippets, or template fragments alongside published pages.

For per-page noindex, use the [`noindex` frontmatter field](/reference/schema/frontmatter).

## `metadata`

A flat record of meta tags injected into every page's `<head>`.

```json
{
  "metadata": {
    "twitter:site": "@tangly",
    "theme-color":  "#EA580C"
  }
}
```

Identical in spirit to `seo.metatags` — present for backward compatibility. Use either.

## `thumbnails`

```json
{
  "thumbnails": {
    "background": "linear-gradient(135deg, #F97316, #C2410C)"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `background` | `string` | Color or gradient for OG thumbnail generation. Reserved — auto-thumbnail generation is on the v0.3 roadmap. |

For now, set page-level OG images explicitly in [frontmatter](/reference/schema/frontmatter#seo) via `seo.ogImage`.
