# `colors`

Five tokens drive every accent and background in the theme. All are hex strings (`#rrggbb`, `#rgb`, or `#rrggbbaa`).

```json
{
  "colors": {
    "primary":    "#EA580C",
    "light":      "#F97316",
    "dark":       "#C2410C",
    "background": { "light": "#FFFFFF", "dark": "#0B0B0F" },
    "anchors":    { "from": "#F97316", "to": "#C2410C" }
  }
}
```

## Tokens

| Field | Type | Default | Description |
|---|---|---|---|
| `primary` | hex | theme | Main accent. Active sidebar link, primary button, focus ring. |
| `light` | hex | theme | Lighter primary tint. Hover states, gradient highlights. |
| `dark` | hex | theme | Darker primary shade. Active states, dark-mode tinted UI. |
| `background` | hex \| `{ light?, dark? }` | theme | Page background. Use the object form to set distinct values per mode. |
| `anchors` | hex \| `{ from?, to? }` | derived from `light`/`dark` | Linear-gradient stops for anchor link styling. |

## Single vs split values

Both `background` and `anchors` accept a single hex (same for both modes) or an object.

```json
"background": "#FAFAF7"
```

â€¦is equivalent to:

```json
"background": { "light": "#FAFAF7", "dark": "#FAFAF7" }
```

## CSS variables emitted

Tangly writes inline CSS variables on `<html>`:

| Variable | Source |
|---|---|
| `--tangly-color-primary` | `colors.primary` |
| `--tangly-color-primary-light` | `colors.light` |
| `--tangly-color-primary-dark` | `colors.dark` |
| `--tangly-color-bg` | `colors.background.light` (or string form) |
| `--tangly-color-bg-dark` | `colors.background.dark` |

Themes consume these in `theme.css`. Override them at the project level (in `theme/styles/theme.css`) to tweak colors without forking a theme.

## Validation

Each hex must match `/^#([0-9a-fA-F]{3}|[0-9a-fA-F]{6}|[0-9a-fA-F]{8})$/`. Schema validation rejects:

- `rgb(...)` / `hsl(...)` â€” Tangly stores raw hex for predictable CSS-var output.
- Named colors (`"red"`, `"blue"`).
- Tailwind tokens (`"sky-500"`).

For named tokens, set them in `theme/styles/theme.css` as CSS variables and reference those from your custom components.

## Defaults from theme

When `colors` is omitted, the active theme's bundled palette kicks in. Tang's defaults:

| Token | Light | Dark |
|---|---|---|
| `primary` | `#EA580C` | `#EA580C` |
| `light` | `#F97316` | `#F97316` |
| `dark` | `#C2410C` | `#C2410C` |
| `background` | `#FAFAF7` | `#0B0B0F` |

Other themes ship their own defaults. See each theme's guide: [tang](/guides/themes/tang), [pith](/guides/themes/pith), [pip](/guides/themes/pip), [readable](/guides/themes/readable), [geist](/guides/themes/geist).
