# `errors`, `banner`

Two unrelated objects that share a page because both deal with content shown outside the normal page flow.

## `errors`

```json
{
  "errors": {
    "404": { "redirect": true }
  }
}
```

### `errors."404"`

| Field | Type | Default | Description |
|---|---|---|---|
| `redirect` | `boolean` | `false` | Redirect 404s to `/` instead of rendering the 404 template. |

### Custom 404 / 500 pages

Drop a `404.mdx` or `500.mdx` at your project root to override the built-in templates. They get the same MDX components as any other page.

```mdx
---
title: Page not found
---

# Lost?

The page you asked for doesn't exist.

[Back to home](/)
```

## `banner`

A dismissible notice rendered above the topbar. Useful for launches, maintenance windows, deprecation notices.

```json
{
  "banner": {
    "id": "v02-launch",
    "type": "info",
    "dismissible": true,
    "content": "Tangly v0.2 ships richer code blocks — <a href=\"/changelog\">see what's new</a>"
  }
}
```

| Field | Type | Default | Description |
|---|---|---|---|
| `content` | `string` | — | Required. HTML allowed (links, basic emphasis). |
| `dismissible` | `boolean` | `false` | Show an X button. |
| `id` | `string` | — | Stable identifier. Used as the `localStorage` dismissal key. Change the id to re-show after a previous dismiss. |
| `type` | `"info" \| "warning" \| "success"` | `info` | Color tone. |

### Resetting the dismissal

Every banner gets dismissed-and-remembered keyed off `id`. To force the banner to reappear after a previous dismissal, change the `id`:

```json
{ "banner": { "id": "v03-launch", "content": "..." } }
```

The browser sees a new id and shows the banner again.
