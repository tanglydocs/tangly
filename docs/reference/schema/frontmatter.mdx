# Frontmatter

Every `.mdx` file may declare frontmatter at the top:

```md
---
title: My page
description: One-line summary
icon: book
tag: Beta
---
```

## Recognized fields

<ParamField path="title" type="string">
  Page title. Falls back to a humanized slug if omitted.
</ParamField>

<ParamField path="sidebarTitle" type="string">
  Override the sidebar entry title (the H1 stays as `title`).
</ParamField>

<ParamField path="description" type="string">
  Subtitle below the H1. Used as the meta description.
</ParamField>

<ParamField path="icon" type="string">
  Lucide or Font Awesome icon name. Shown in sidebar + cards that link here.
</ParamField>

<ParamField path="tag" type="string">
  Small badge in the sidebar (e.g. "Beta", "New", "Phase 2").
</ParamField>

<ParamField path="api" type="string">
  Manual narrative for an OpenAPI endpoint (Phase 3).
</ParamField>

<ParamField path="openapi" type="string">
  `METHOD path` to auto-render an endpoint.
</ParamField>

<ParamField path="keywords" type="string[]">
  Custom meta keywords.
</ParamField>

<ParamField path="noindex" type="boolean">
  Exclude from sitemap.xml + llms.txt + search index.
</ParamField>

<ParamField path="mode" type="enum">
  `default | wide | center | custom`. Hides the right TOC at `wide`/`center`/`custom`.
</ParamField>

<ParamField path="draft" type="boolean">
  Phase 2: hidden in build, visible in dev with badge.
</ParamField>

<ParamField path="template" type="string">
  Phase 2: render the page through `./templates/<name>.astro`.
</ParamField>

<ParamField path="aiContext" type="string">
  Phase 6: extra context sent to the AI assistant when answering on this page.
</ParamField>

## Custom fields

Any extra field is passed through to the active template via `frontmatter`. Useful for landing-page templates that consume `hero`, `cta`, etc.
