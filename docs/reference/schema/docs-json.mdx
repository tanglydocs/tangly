# `docs.json`

`docs.json` lives at your project root. It configures everything that isn't page content.

The full Zod schema is in [`packages/schema/src/docs-json.ts`](https://github.com/tanglydocs/tangly/blob/main/packages/schema/src/docs-json.ts). The published JSON Schema (at `https://tangly.dev/schema/docs.json` and `node_modules/tangly/schema/docs.json`) is what your editor uses for autocomplete and validation.

## Editor support

Add `$schema` to the top of your file and any editor with JSON Schema support — VS Code, Cursor, JetBrains, Zed — gets autocomplete and inline validation:

```json
{
  "$schema": "https://tangly.dev/schema/docs.json",
  "name": "My docs"
}
```

`tangly migrate` rewrites `$schema` to the canonical URL automatically.

## Core fields

These three live on this page; everything else is split into a dedicated reference.

<ParamField path="name" type="string" required>
  Display name. Used in the topbar logo, OG title, and meta tags. Minimum one character.
</ParamField>

<ParamField path="description" type="string">
  Default meta description. Per-page values from frontmatter override.
</ParamField>

<ParamField path="$schema" type="string">
  JSON Schema URL for editor autocomplete. `tangly migrate` rewrites this to the canonical URL.
</ParamField>

<ParamField path="appearance" type="object">
  Light/dark mode behavior + page chrome.

  - `default: "light" | "dark" | "system"` — initial theme. Default `"system"`.
  - `strict: boolean` — disables the user's theme toggle. Forces `default`.
  - `readingTime: boolean` — show estimated reading time on every page (default `false`).
  - `readingProgress: boolean` — show the 2px scroll-progress bar across the top (default `false`).
</ParamField>

<ParamField path="search" type="object">
  - `prompt: string` — placeholder text in the search modal. Default `"Search docs..."`.
</ParamField>

## Strict parsing

```ts
import { parseDocsJson, safeParseDocsJson } from "@tanglydocs/schema";

const cfg = parseDocsJson(JSON.parse(raw));        // throws on error
const result = safeParseDocsJson(JSON.parse(raw)); // discriminated result
```

Use `safeParseDocsJson` from CI scripts so you can format errors yourself.

## Passthrough

`docs.json` uses Zod's `passthrough()` — unknown top-level keys are preserved on the parsed object so projects mid-migration with extra fields don't fail validation. Tangly only enforces the shape of fields it knows about.
