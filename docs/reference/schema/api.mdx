# `api`

The `api` object configures every OpenAPI surface in the docs site: which spec(s) to load, which viewer to render with, what auth method the try-it-out form should use.

```json
{
  "api": {
    "baseUrl": "https://api.example.com",
    "auth":    { "method": "bearer", "name": "Authorization" },
    "playground": { "mode": "interactive" },
    "openapi": ["./openapi.json"],
    "viewer":  "tangly",
    "asyncapi": "./asyncapi.yaml",
    "mdx":     { "server": "./server.mdx" }
  }
}
```

## Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `baseUrl` | `string \| string[]` | from spec | Default base URL for try-it-out. Array enables a multi-environment switcher (prod/staging/dev). |
| `auth.method` | `"bearer" \| "basic" \| "key" \| "none"` | `none` | Auth scheme for the try-it-out form. |
| `auth.name` | `string` | ‚Äî | Header / query parameter name when `method="key"`. |
| `playground.mode` | `"interactive" \| "simple" \| "hide"` | `interactive` | Try-it-out UI variant. `hide` removes the form entirely. |
| `playground.proxy` | `boolean` | `false` | Reserved. Proxy try-it-out through Tangly's edge to bypass CORS. Currently a no-op. |
| `openapi` | `string \| string[]` | ‚Äî | Spec URL(s) or local file path(s). |
| `viewer` | `"tangly" \| "scalar" \| "redoc" \| "stoplight"` | `tangly` | Rendering engine for auto-generated OpenAPI pages. |
| `asyncapi` | `string \| string[]` | ‚Äî | AsyncAPI spec(s). |
| `mdx.server` | `string \| string[]` | ‚Äî | Server-side MDX. Reserved for the upcoming chat integration. |

## Auth methods

```json
{ "auth": { "method": "bearer" } }
```

Renders a "Bearer token" field above the try-it-out form.

```json
{ "auth": { "method": "basic" } }
```

Renders user + password fields.

```json
{ "auth": { "method": "key", "name": "X-Api-Key" } }
```

Renders a single token field that gets sent as the named header.

```json
{ "auth": { "method": "none" } }
```

No auth UI. Default.

<Note>The `cookie` method is intentionally absent. Browser fetch can't set the `Cookie` header from user-supplied values. Same-origin requests already get the browser's cookies via `credentials: "include"` ‚Äî exposed via the try-it-out form's "session" toggle, not as a separate auth mode.</Note>

## Picking a viewer

```json
{ "api": { "viewer": "scalar" } }
```

Auto-generated OpenAPI pages then use [`<OpenApiScalar>`](/reference/components/openapi-scalar) instead of [`<OpenApiEndpoint>`](/reference/components/openapi-endpoint). Switch any time without changing your specs.

| `viewer` | Component | Notes |
|---|---|---|
| `tangly` (default) | [`<OpenApiEndpoint>`](/reference/components/openapi-endpoint) | Tangly's compact built-in. Lightest weight. |
| `scalar` | [`<OpenApiScalar>`](/reference/components/openapi-scalar) | [Scalar](https://scalar.com)'s embed. |
| `redoc` | [`<OpenApiRedoc>`](/reference/components/openapi-redoc) | [ReDoc](https://redocly.com/redoc) standalone. |
| `stoplight` | [`<OpenApiStoplight>`](/reference/components/openapi-stoplight) | [Stoplight Elements](https://stoplight.io/open-source/elements). |

## Multi-environment baseUrl

```json
{
  "api": {
    "baseUrl": [
      "https://api.example.com",
      "https://staging.api.example.com",
      "http://localhost:3000"
    ]
  }
}
```

Renders an environment switcher in the try-it-out form. Selection is sticky per visitor.
