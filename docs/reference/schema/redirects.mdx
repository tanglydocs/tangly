# `redirects`

Array of `{ source, destination, permanent? }`. Tangly emits HTML `<meta http-equiv="refresh">` files at `dist/<source>/index.html` so any static host serves the redirect without server-side configuration.

```json
{
  "redirects": [
    { "source": "/old-guide", "destination": "/guides/new", "permanent": true },
    { "source": "/legacy",    "destination": "/changelog",  "permanent": false }
  ]
}
```

## Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `source` | `string` | — | Path to redirect from. Required. Leading slash required. |
| `destination` | `string` | — | Where to send the request. Required. Internal slug or external URL. |
| `permanent` | `boolean` | `false` | When `true`, the meta-refresh page also emits SEO-friendly markers. Static hosts that respect `_redirects` (Netlify, Cloudflare Pages) translate this to a 301. |

## Adapter-specific behavior

- **Static** (default): emits `<source>/index.html` with `<meta http-equiv="refresh">`. Works on any host.
- **Vercel** (`tangly build --adapter vercel`): writes a `vercel.json` `redirects` block alongside the meta-refresh fallback.
- **Cloudflare** (`tangly build --adapter cloudflare`): emits `_redirects` with status code (`301` for permanent, `302` otherwise).
- **Netlify**: same as Cloudflare via `_redirects`.

## Cycle detection

`tangly check` flags loops at config-time. If `/a` redirects to `/b` and `/b` redirects to `/a`, the build fails with both source files named in the error.

## Subpath hosting

Redirects honor the `--base` flag. `tangly build --base /docs` rewrites `source: "/old"` to `/docs/old`. See [Subpath hosting](/guides/deploying/subpath-hosting).
