# Themes

Tangly ships two themes:

- **Tang** — default, Mintlify-Mint inspired.
- **Pith** — editorial alternative, denser, serif headings (Phase 2).

Switch via `docs.json`:

```json
{ "theme": "pith" }
```

## Color tokens

The `colors` field in `docs.json` becomes CSS variables:

```css
:root {
  --tangly-color-primary: #0ea5e9;
  --tangly-color-bg: #ffffff;
  --tangly-color-fg: #0f172a;
  --tangly-color-muted: #64748b;
  --tangly-color-border: #e2e8f0;
}
.dark {
  --tangly-color-primary: #38bdf8;
  --tangly-color-bg: #0b1120;
  --tangly-color-fg: #e2e8f0;
}
```

These flow through every component — buttons, links, code blocks, sidebar active state.

## Light + dark

Tangly auto-detects `prefers-color-scheme` and provides a manual toggle in the top-bar.

## Component shadowing (Phase 2)

Drop a file at `./components/Note.tsx` (or `.astro`) in your project to override the default `Note` without forking the theme:

```
my-docs/
├── docs.json
├── components/
│   └── Note.astro    ← user override
└── introduction.mdx  ← MDX still uses <Note>; resolves to your file
```

Resolution order:

1. `./components/<Name>.{tsx,astro}` in user project
2. Active theme's component
3. Built-in default

The shadow component must export a default that accepts the same props as the original.

## Custom themes (Phase 2+)

Themes are npm packages with a fixed shape. To author your own:

```
packages/theme-mine/
├── src/
│   ├── Layout.astro
│   ├── Sidebar.astro
│   ├── TopNav.astro
│   ├── Footer.astro
│   ├── PageShell.astro
│   ├── styles/
│   │   ├── theme.css
│   │   └── prose.css
│   └── theme.config.ts
└── package.json
```

Then in `docs.json`:

```json
{ "theme": "mine" }
```

Tangly looks for `@tangly/theme-mine` first, then any package whose name ends in `theme-mine`.
